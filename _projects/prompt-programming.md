Prompt Programming is a learning platform for dialogue-based computational problem solving with generative AI models.

The platform supports multi-turn prompting, problems with multiple interdependent functions, and on-request code execution. It also supports direct code editing in the same workflow, so students can move between prompting and targeted code changes while solving tasks.

<img class="project-detail-image" src="/assets/img/projects/prompt-programming.png" alt="Prompt Programming interface screenshot" />

Across our studies, learners solve programming tasks through iterative, multi-turn workflows that combine prompting and code editing. Prompting is often used to generate and restructure candidate solutions, while editing is used to make precise corrections and finalize details. As task complexity increases, sessions become more interactive and include more targeted edits. Students also use execution and checking strategically to validate intermediate ideas before moving on. We also observe clear differences across performance groups, with higher-performing students tending to rely more on prompt-driven decomposition and lower-performing students tending to rely more on direct code edits.

Related work includes [Interleaving Natural Language Prompting with Code Editing for Solving Programming Tasks with Generative AI Models](https://arxiv.org/abs/2509.14088) (ITiCSE 2026, to appear) and *Say What? Examining Text and Voice Input Modalities for Prompt-Based Programming in Computing Education* (ITiCSE 2026, to appear). The project was also presented at [Dagstuhl Seminar 25311](https://www.dagstuhl.de/25311).

You can read the main paper on [ACM Digital Library](https://doi.org/10.1145/3724363.3729094), access the preprint on [arXiv](https://arxiv.org/abs/2503.04267), and try the system on the [Prompt Programming website](https://www.promptprogram.org).
